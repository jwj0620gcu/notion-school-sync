# 📓 Daily Snippet Automation

> 노션에 일기를 쓰면, AI가 자동으로 정형화해서 1000.school에 제출합니다.

---

## 왜 만들었나

**Daily Snippet을 쓰는 건 좋은데, 매일 형식에 맞게 정리해서 올리는 게 번거롭다.**

```
기존 흐름
  노션에 오늘 한 일 메모
      → 형식에 맞게 직접 다듬기
          → 1000.school에 복붙
              → AI 채점 버튼 클릭
                  → 저장

문제점
  - 하루도 빠짐없이 반복되는 단순 작업
  - 형식이 틀리거나 빠진 항목이 생김
  - 바쁜 날은 건너뛰게 됨
```

**→ 노션에 메모만 해두면 나머지는 전부 자동으로 처리되게 만들었다.**

---

## 한 줄 요약

**노션 작성 → Gemini AI 자동 다듬기 → 1000.school 자동 제출 → 주간·월간 AI 감독 리포트**

## 웹 서비스 구성

```
브라우저 (Vercel · Next.js)
    │  구글 로그인 / API 키 입력 / 동작 상태 확인
    │
    ▼
백엔드 (Railway · FastAPI)
    │  유저별 API 키 암호화 저장 / 스케줄러 실행
    │
    ▼
DB (Supabase · PostgreSQL)
    │  유저 설정 / 스니펫 / 분석 결과 저장
```

| 구성 | 역할 |
|------|------|
| **Vercel (Next.js)** | 로그인·설정·대시보드 UI |
| **Railway (FastAPI)** | API 서버 + 스케줄러 (10분마다 전체 유저 실행) |
| **Supabase** | Google OAuth + PostgreSQL DB |
| **Notion API** | 페이지 변경 감지, 리포트 페이지 자동 생성 |
| **1000.school API** | 일간·주간 스니펫 자동 제출 |
| **Gemini 2.5 Flash** | 스니펫 다듬기 + 주간·월간 AI 분석 |

## 핵심 기능

### 1. 노션 변경 감지 → AI 자동 다듬기 → 제출

```
매 10분 체크
    │
    ├─ 변경 없음 → 대기
    │
    └─ 변경 감지
            │
            ├─ 빈 템플릿 → 스킵 (내용 작성 후 다음 루프)
            │
            └─ 내용 있음 → Gemini 다듬기 → 1000.school 제출
```

**다듬기 전/후 예시**

| | 내용 |
|--|------|
| **원문** | `회의 했음. API 연동. 책 읽음.` |
| **다듬기 후** | 아래 참고 ↓ |

```
## 오늘 한 일
- 팀 회의 참여 (주요 안건 논의 및 역할 분담 완료)
- NotionAPI 연동 작업 (스니펫 자동 업로드 흐름 설계)
- 파타고니아 인사이드 독서 (120p까지)

## 수행 목적
- 팀 방향성 재정렬 및 이번 주 우선순위 확정
- 자동화로 반복 작업 제거, 기록 품질 향상

## 하이라이트
- API 연동 구조 설계 완료로 다음 단계 구현이 명확해짐

## 로우라이트
- 회의 시간이 길어져 개인 작업 시간 부족

## 내일의 우선순위
- NotionAPI: 스니펫 업로드 엔드포인트 구현 완료

## 헬스 체크 (10점)
- 7/10 (전반적으로 양호)
```

### 2. 주간 리포트 (매주 월요일 오전 9시 자동)

```
월요일 오전 9시
    │
    ├─ 1) DB에서 지난주(월~일) 일간 스니펫 조회
    ├─ 2) Gemini 7지표 분석 (1회 호출)
    ├─ 3) 주간 스니펫 작성 → 1000.school 업로드
    └─ 4) 노션 주간 리포트 페이지 자동 생성
```

**노션 자동 생성 구조**

```
Daily Snippet (부모 페이지)
├─ 2026-03                 ← 월별 컨테이너 자동 생성
│   ├─ 2026-03-22
│   └─ 2026-03-23
├─ 📅 주간 리포트
│   ├─ W12 (03/16~03/22)   ← 매주 새 페이지
│   └─ W13 (03/23~03/29)
└─ 📆 월간 리포트
    └─ 2026-03 (3월)       ← 매달 새 페이지
```

> 부모 페이지만 연결해두면 모든 하위 페이지는 자동으로 생성됩니다.

### 3. 7대 AI 감독 지표

매주·매달 스니펫 전체를 분석해 7가지 지표로 평가합니다.

| 지표 | 무엇을 보나 | 높으면 |
|------|------------|--------|
| 🔥 번아웃 위험도 | 헬스 점수 저하, 피로 패턴 반복 | 위험 ⚠️ |
| 👥 팀 건강도 | 팀 기여 빈도, 갈등 조짐 | 좋음 ✅ |
| 💪 성실도 | 작성 연속성, 우선순위 달성률 | 좋음 ✅ |
| 🔁 문제 재발성 | 동일 로우라이트 반복 여부 | 재발 많음 ⚠️ |
| 🧠 성장 지수 | 새 학습, 피드백 이행 여부 | 좋음 ✅ |
| ⚡ 실행 집중도 | 목표와 실제 행동 일치도 | 좋음 ✅ |
| 💬 감정 에너지 | 글의 긍정도, 자기효능감 변화 | 긍정적 ✅ |

**토큰 절약 구조**

```
스니펫 원문 전체 (~1,000자 × N일)  →  직접 보내면 과금 폭발 💥
    ▼
핵심 필드만 압축 (~100자/일)
날짜 | 헬스 | 하이라이트 | 로우라이트 | 내일 목표
    ▼
Gemini 전송 → 7지표 분석
```

## 시작하기

### 웹 서비스 사용 (권장)

1. 웹 접속 → 구글 로그인
2. Settings 페이지에서 API 키 입력
   - Notion Integration 토큰
   - Notion 부모 페이지 ID
   - 1000.school API 키
   - Gemini API 키 (무료: [aistudio.google.com](https://aistudio.google.com/app/apikey))
3. 완료 → 이후 자동 실행

### 로컬 실행

```bash
git clone https://github.com/leten02/notion-school-sync.git
cd notion-school-sync
bash setup.sh        # 가상환경 + 패키지 + DB 자동 세팅
cp .env.example .env # API 키 입력
source venv/bin/activate
python main.py --watch
```
