# 📋 Notion × 1000.school 자동 동기화 + AI 감독 리포트

> 노션에 데일리 기록을 작성하면, AI가 다듬어서 자동으로 제출하고 흐름을 분석해줍니다.

---

## 한 줄 요약

**노션 작성 → AI 정형화 → 1000.school 자동 제출 → AI 감독 리포트 생성**

---

## 전체 흐름

```
📝 노션에 오늘 내용 작성
        │
        ▼  (변경 감지 시 즉시)
📤 원문 그대로 1000.school 업로드
        │
        ▼  (Tomorrow 섹션 작성 + 30분 무변경)
✨ Gemini AI가 내용 정형화 (polish)
        │
        ▼
📤 다듬어진 최종본으로 덮어쓰기
        │
        ▼  (매일 오전 9시)
📊 AI 감독 리포트 자동 갱신
```

---

## 주요 기능

### 1. 자동 변경 감지 & 업로드

- **10분마다** 노션 페이지의 수정 여부를 확인합니다.
- 변경이 감지되면 **원문 그대로 즉시** 1000.school에 업로드합니다.
- 내용이 비어있으면 업로드를 건너뜁니다.

### 2. Gemini AI 자동 다듬기 (Polish)

작성이 끝났다고 판단되면 AI가 내용을 정형화해 최종본으로 덮어씁니다.

**Polish가 실행되는 조건 (2가지)**

| 조건 | 설명 |
|------|------|
| ✅ **Tomorrow 섹션 + 30분** | Tomorrow(내일 우선순위)를 작성하고 30분간 추가 수정이 없으면 "작성 완료"로 판단 |
| ⏰ **8:50 폴백** | Tomorrow를 못 적었더라도 오전 8:50~8:59 사이에 내용이 있으면 강제 실행 (9시 데드라인 전 보장) |

> 하루에 단 **1번만** 실행됩니다. 한 번 polish가 완료되면 당일 추가 실행은 없습니다.

**다듬기 결과 예시**

원문:
```
오늘 회의 했음. API 연동 작업. 책 읽음.
```

Gemini 다듬기 후:
```
## 오늘 한 일
- 팀 회의 참여 (주요 안건 논의 및 역할 분담)
- NotionAPI 연동 작업 착수 (스니펫 업로드 흐름 설계)
- 파타고니아 인사이드 독서 (120페이지까지)

## 하이라이트
- API 연동 구조 설계 완료로 다음 단계 구현이 명확해짐

## 로우라이트
- 회의 시간이 예상보다 길어져 개인 작업 시간 부족

## 내일의 우선순위
- NotionAPI: 스니펫 업로드 엔드포인트 구현 완료
- 독서: 다음 챕터 읽기

## 헬스 체크 (10점)
- 7/10 (전반적으로 양호)
```

### 3. 역방향 동기화 (1000.school → 노션)

1000.school에 피드백이 달리거나 외부에서 수정된 내용을 노션에 자동 반영합니다.

**깜빡임 방지 로직**

```
업로드 성공 → 업로드한 내용의 해시값 저장
                    │
                    ▼  (10분마다)
      1000.school 현재 내용 해시 계산
                    │
          ┌─────────┴─────────┐
      같으면                 다르면
      스킵 ✅               노션 업데이트 ✅
   (깜빡임 없음)        (피드백 반영)
```

> 핵심: 업로드된 내용 기준으로 해시를 저장하기 때문에, 동일한 내용이면 절대 덮어쓰지 않습니다.

### 4. 매일 오전 9시 자동 실행

| 작업 | 설명 |
|------|------|
| 📄 오늘 페이지 생성 | 오늘 날짜 노션 페이지를 템플릿과 함께 자동 생성 |
| 🔄 전날 최종 동기화 | 어제 스니펫 최종본을 노션에 반영 |
| 📊 감독 리포트 갱신 | AI 감독 리포트 하루 1회 자동 업데이트 |

> 페이지 생성 실패 시 다음 루프(10분 후)에 자동 재시도합니다.

### 5. AI 감독 리포트

전체 스니펫 흐름을 분석해 노션 리포트 페이지에 작성합니다.

**7개 분석 지표**

| 지표 | 설명 | 점수 기준 |
|------|------|------|
| 🔥 번아웃 위험도 | 헬스 점수 저하·피로 패턴 감지 | 높을수록 위험 |
| 👥 팀 건강도 | 팀 기여·갈등 조짐 분석 | 높을수록 좋음 |
| 💪 성실도 | 작성 연속성·우선순위 달성률 | 높을수록 좋음 |
| 🔁 문제 재발성 | 동일 로우라이트 반복 여부 | 높을수록 재발 많음 |
| 🧠 성장 지수 | 새로운 학습·피드백 이행 여부 | 높을수록 좋음 |
| ⚡ 실행 집중도 | 목표와 실제 행동 일치도 | 높을수록 좋음 |
| 💬 감정 에너지 | 글의 온도·자기효능감 변화 | 높을수록 긍정 |

**토큰 절약 구조**

```
스니펫 전체 원문 (날짜당 ~1000자)
        │
        ▼  DB에서 핵심만 추출
압축 요약본 (~100자/일) → Gemini 전송
        │
        ▼
7개 지표 분석 결과 → 노션 리포트
```

## 노션 데일리 템플릿

매일 오전 9시에 생성되는 기본 템플릿 구조:

```
## What          ← 오늘 한 일
## Why           ← 수행 목적
## Team Value Added  ← 팀 기여
## Highlight     ← 잘된 점
## Lowlight      ← 아쉬운 점
## Tomorrow      ← 내일 우선순위 (★ polish 트리거 역할)
```

---

## 데이터 흐름 & 저장 구조

```
노션 페이지
    │  get_notion_content()
    ▼
원문 텍스트
    │  run_once(polish=False) → 즉시
    │  run_once(polish=True)  → Tomorrow+30분 or 8:50
    ▼
1000.school API (POST/PUT)
    │
    ├──▶ SQLite DB 저장
    │       snippets 테이블: content, health_score, highlights 등
    │       analysis 테이블: 7개 지표 점수, 분석 결과
    │
    └──▶ 역방향 동기화 (해시 비교, 10분마다)
              변경 있을 때만 노션 반영
```

## 설치 및 설정

### 1. 의존성 설치

```bash
pip install -r requirements.txt
```

### 2. 환경 변수 설정 (`.env`)

```env
NOTION_TOKEN=secret_xxxx       # 노션 Integration 토큰
NOTION_PAGE_ID=xxxx            # 데일리 페이지 부모 페이지 ID
SCHOOL_API_KEY=I_wrdo_xxxx     # 1000.school API 키
GEMINI_API_KEY=AIzaSy_xxxx     # Gemini API 키
```

**노션 Integration 설정**
1. [notion.so/my-integrations](https://www.notion.so/my-integrations) 에서 Integration 생성
2. 부모 페이지 우측 상단 **···** → **연결** → Integration 추가

**Gemini API 키 발급**
- [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)

### 3. DB 초기화

```bash
python db.py
```

## 실행 방법

```bash
# Watch 모드 시작 (메인 기능)
python main.py --watch

# 백그라운드로 실행 (터미널 꺼도 유지)
nohup python main.py --watch > watch.log 2>&1 &

# 실행 중인지 확인
ps aux | grep main.py

# 종료
kill $(ps aux | grep 'main.py --watch' | grep -v grep | awk '{print $2}')

# 감독 리포트 수동 갱신
python main.py --report

# 오늘 템플릿 수동 추가
python main.py --template

# 과거 스니펫 전체 노션 동기화
python sync_to_notion.py
```
